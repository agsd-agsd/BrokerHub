# BrokerHub 启动指南

以下命令默认都在当前仓库根目录执行：

```powershell
cd C:\Users\ASUS\Desktop\BrokerHub
```

## 1. 主程序编译

Windows 推荐先编译固定 exe：

```powershell
go build -o blockEmulator_Windows_Precompile.exe .
```

如果是在 Linux 环境，可编译为：

```bash
go build -o blockEmulator_Linux_Precompile .
```

## 2. 生成启动脚本并运行

`-g` 会生成多节点启动脚本，不会直接跑仿真。  
生成完成后，再执行生成出来的 `.bat` 或 `.sh`。

### 2.1 `taxrate` 模式

```powershell
go run . -g -N 4 -S 2 -m 4 --exchange_mode limit300 --fee_optimizer taxrate --sim_seed 0
.\bat_shardNum=2_NodeNum=4_mod=Broker_b2e.bat
```

### 2.2 `paper_monopoly` 模式

```powershell
go run . -g -N 4 -S 2 -m 4 --exchange_mode limit300 --fee_optimizer paper_monopoly --sim_seed 42
.\bat_shardNum=2_NodeNum=4_mod=Broker_b2e.bat
```

Linux 下对应命令：

```bash
go run . -g -N 4 -S 2 -m 4 --exchange_mode limit300 --fee_optimizer paper_monopoly --sim_seed 42
sh ./bat_shardNum=2_NodeNum=4_mod=Broker_b2e.sh
```

## 3. 常用参数

- `--exchange_mode limit300`：推荐，最多运行 300 个 epoch
- `--exchange_mode limit100`：短跑测试
- `--exchange_mode infinite`：持续运行
- `--fee_optimizer taxrate`：默认费率优化模式
- `--fee_optimizer paper_monopoly`：论文风格 monopoly fee control 模式
- `--sim_seed 0`：非固定随机
- `--sim_seed > 0`：可复现实验

## 4. 输出位置

- Hub CSV：`.\hubres\hub0.csv`、`.\hubres\hub1.csv`
- 绘图 PDF：`.\hubres\AsterYao_brokerhub_combined_metrics.pdf`、`.\hubres\AsterYao_brokerhub_extended_metrics.pdf`
- 日志目录：`.\log`

## 5. 绘图脚本启动命令

主图：

```powershell
python .\draw_diff_hub.py
```

扩展图：

```powershell
python .\draw_diff_hub_extended.py
```

两份绘图脚本都会读取 `.\hubres\hub0.csv` 和 `.\hubres\hub1.csv`，并把输出 PDF 写回 `.\hubres\`。

## 6. 用 Git 和 GitHub 追踪每次实验数据

推荐不要只覆盖同一份 `hub0.csv` / `hub1.csv`，而是在每次实验结束后归档一份独立快照。

### 6.1 归档当前实验输出

如果当前输出在默认目录 `.\hubres\`：

```powershell
python .\archive_experiment.py --name paper-monopoly-seed42 --command "go run . -g -N 4 -S 2 -m 4 --exchange_mode limit300 --fee_optimizer paper_monopoly --sim_seed 42"
```

如果当前输出在本地 harness 目录 `.\supervisor\committee\hubres\`：

```powershell
python .\archive_experiment.py --source-dir .\supervisor\committee\hubres --name local-harness-seed42 --command "go test ./supervisor/committee -run TestRunPaperMonopolyLocalHarnessLimit300Seed42 -v"
```

归档后会生成一个新目录：

```text
.\experiments\runs\YYYYMMDD-HHMMSS-<name>\
```

里面会包含：

- `artifacts\`：CSV、PDF 等实验产物副本
- `metadata.json`：归档时间、分支、commit、命令、备注
- `README.md`：这一轮实验的人类可读摘要

### 6.2 提交到 Git 和 GitHub

```powershell
git add .\experiments\runs\
git commit -m "exp: paper_monopoly seed42"
git push
```

建议约定：

- 每次实验都归档成一个新目录，不覆盖旧实验
- commit message 里带上实验名、optimizer、seed、核心参数
- 只把最终要保留的实验快照推到 GitHub；临时调参输出不要混进同一个 commit
